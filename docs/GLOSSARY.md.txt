# Glossary

## Maven : A build automation and dependency management tool for Java projects.
## POM : Project Object Model. The XML file that tells Maven how to build the project.
## Dependency : An external library your application uses. Example: Spring Web
## Artifact : The output produced by Maven. Usually a JAR file.
## Snapshot : A development version of an application.
## Starter Dependency : A Spring Boot dependency that brings in a collection of related libraries. Example: spring-boot-starter-web
## Lombok : A library that generates boilerplate code such as getters, setters, constructors, builders, and more during compilation.
## DevTools : A Spring Boot library that automatically restarts the application when code changes during development.
## Bean : A Java object managed by the Spring IoC Container.
## IoC (Inversion of Control) : Spring manages object creation instead of the developer.
## Dependency Injection : Spring automatically provides required objects (dependencies) to other objects.
## Component Scan : Spring searches packages for annotated classes and registers them as beans.
## Embedded Tomcat : A web server bundled inside the Spring Boot application.
## ORM : Object Relational Mapping. Maps Java objects to relational database tables.
## Entity : A Java class mapped to a database table.
## JPA : Java Persistence API. A specification for ORM.
## Hibernate : The default JPA implementation used by Spring Boot.
## ddl-auto : Hibernate property that controls schema generation.
Options:
- none
- validate
- update
- create
- create-drop
## Repository : Data Access Layer responsible for communicating with the database.
## JpaRepository : Spring Data JPA interface that provides CRUD operations without writing SQL.
## Service Layer : Contains business logic.
## Dependency Injection (DI) : Spring automatically provides required dependencies instead of creating them manually.
## Constructor Injection : Preferred way of injecting dependencies in Spring Boot.
## REST Controller : A Spring component that exposes REST APIs and automatically converts Java objects into JSON.
## Request Mapping : Maps HTTP URLs to controller methods.
## Get Mapping : Handles HTTP GET requests.
## @RequestBody: Converts JSON from an HTTP request into a Java object.
## @PostMapping: Maps HTTP POST requests to controller methods.
## Optional : A Java container object that may or may not contain a value.
## @PathVariable : Extracts values from the URL into method parameters.
## RuntimeException: An unchecked exception that can be thrown during application execution.
## @RestControllerAdvice : Handles exceptions globally for all REST controllers.
## @ExceptionHandler : Maps specific exceptions to handler methods.
## Bean Validation : A standard Java mechanism for validating object fields using annotations.
## @Valid : Triggers validation on a request object.
## @NotBlank : Ensures a String is not null and not empty after trimming whitespace.
## @Positive : Ensures a numeric value is greater than zero.
## @Min : Specifies the minimum allowed numeric value.
## @ControllerAdvice : Provides centralized exception handling across all controllers.
## MethodArgumentNotValidException : Thrown when a request object annotated with @Valid fails validation.
## DTO (Data Transfer Object) : An object used to transfer data between application layers without exposing the underlying database entity.
## Request DTO : Receives data from the client.
## Response DTO : Sends controlled data back to the client.
## Mapper : A dedicated class responsible for converting one object type into another.
## Manual Mapping : Explicitly copying values from one object to another without using a framework.
## Stream : A Java API for processing collections in a functional and declarative style.
## map() : Transforms each element in a stream into another object.
## Method Reference : A concise syntax for referring to an existing method, e.g., ProductMapper::toResponseDTO.
