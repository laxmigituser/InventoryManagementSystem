# Software Architecture

## Overview

The Inventory Management System follows a three-tier architecture.

```
+----------------------+
|      Frontend        |
| React + Material UI  |
+----------+-----------+
           |
           | HTTP/JSON
           |
+----------v-----------+
|       Backend        |
| Spring Boot REST API |
+----------+-----------+
           |
           | JPA
           |
+----------v-----------+
|      PostgreSQL      |
|      Database        |
+----------------------+
```

---

## Responsibilities

### Frontend

Responsible for:

- Displaying screens
- Capturing user input
- Calling backend APIs
- Showing results

The frontend never talks directly to the database.

---

### Backend

Responsible for:

- Business logic
- Validation
- Authentication
- Authorization
- Database operations

---

### Database

Responsible for:

- Storing application data
- Maintaining relationships
- Ensuring data integrity

---
