# Backend Fundamentals

## Client: Software that sends requests to the backend.
Examples:
- Browser
- Postman
- Playwright

## Server: Software that receives requests and sends responses.
Our server: Spring Boot

## HTTP: Protocol used for communication between client and server.
## HTTP Methods
GET: Read data
POST: Create data
PUT:  Update data
DELETE: Delete data

## JSON: Format used to exchange data between frontend and backend.

## REST API: Collection of HTTP endpoints exposed by the backend.

# JPA & Hibernate
## ORM : Object Relational Mapping converts Java objects into database records.
## JPA : Java Persistence API . Specification only.
## Hibernate : Implementation of JPA. Spring Boot uses Hibernate by default.
## Entity : Represents a database table.
Annotations learned:
- @Entity
- @Table
- @Id
- @GeneratedValue
- @Column

## Product Entity
Entity Name : Product
Mapped Table: products
Primary Key: id
Auto Generation: GenerationType.IDENTITY
Data Types:
- String
- BigDecimal
- Integer
- Long

# Repository Layer : Repository is responsible for database communication.
Technology:
- Spring Data JPA
Interface:ProductRepository
Extends: JpaRepository<Product, Long>
Benefits:
- No SQL required
- CRUD methods generated automatically

# Service Layer
Purpose: Contains business logic.
Annotation:@Service
Dependency Injection : Constructor Injection
Benefits:
- Separation of concerns
- Easier testing
- Follows SRP
- Business rules centralized

## DTO Layer
Created DTO package.
Added:
- ProductRequestDTO

- toEntity()
